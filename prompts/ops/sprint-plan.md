# Build a Task or Sprint Plan

**Use case:** Break down a week or two-week sprint into prioritized, specific tasks across multiple projects.
**Works with:** Claude (any version)

---

Build a sprint plan for the next [1 WEEK / 2 WEEKS].

Projects in flight: [LIST ACTIVE PROJECTS]
Goals this sprint: [LIST 3–5 GOALS]

Break each goal into specific tasks. Assign priority:
- **P1**: must ship this sprint
- **P2**: should ship this sprint
- **P3**: stretch goal, ship if time allows

Flag any blockers (things that can't start until something else is done) and dependencies (things that need input from another person or system).

Output as a table: Task | Project | Priority | Estimated effort | Blocker (if any)

**How to use P1/P2/P3:**
P1 items are the only ones that matter if the sprint gets compressed. If there are more than 3–4 P1 items, you are over-committing. Be honest about what is truly must-ship vs. would-be-nice.
