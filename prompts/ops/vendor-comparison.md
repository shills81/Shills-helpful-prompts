# Build a Vendor or Tool Comparison

**Use case:** Evaluate 2–4 tools or vendors side by side to make a stack or procurement decision.
**Works with:** Claude (any version)

---

Compare [LIST 2–4 TOOLS OR VENDORS] for [DESCRIBE THE USE CASE].

Evaluation criteria:
1. [CRITERION 1, e.g. pricing]
2. [CRITERION 2, e.g. ease of integration]
3. [CRITERION 3, e.g. scalability]
4. [CRITERION 4, e.g. support quality]
5. [CRITERION 5, e.g. deal-breakers]

For each tool: pros, cons, pricing summary, and any deal-breakers.

Conclude with a recommendation and a one-sentence reason why.

Output: comparison table + recommendation paragraph.

**The deal-breaker rule:**
If a tool has a deal-breaker (missing a required feature, pricing that doesn't fit, vendor lock-in you can't accept), no amount of pros in other categories overcomes it. State deal-breakers clearly rather than averaging them into the score.
