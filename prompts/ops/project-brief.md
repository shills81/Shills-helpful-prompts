# Write a Project Brief or Spec

**Use case:** Define the scope, deliverables, and success criteria for a project before any work begins.
**Works with:** Claude (any version)

---

Write a project brief for [PROJECT NAME].

Include:
- **Objective** (one sentence): what this project achieves
- **Background**: why this, why now
- **Scope**: what is explicitly IN and what is explicitly OUT
- **Deliverables**: specific and measurable (not "a dashboard" but "a single-page HTML dashboard that loads in under 2 seconds and shows X, Y, Z")
- **Timeline**: key milestones and final deadline
- **Dependencies**: what this project needs from other people or systems
- **Success criteria**: how we will know it worked

Max 1 page.

**Why the out-of-scope section matters:**
The clearest signal of a poorly scoped project is a missing out-of-scope list. If you don't write it down, scope will creep into it. The out-of-scope section protects you and sets accurate expectations.
