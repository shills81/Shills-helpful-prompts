# Write an SOP or Runbook

**Use case:** Document a repeatable process so anyone can execute it correctly without prior knowledge.
**Works with:** Claude (any version)

---

Write an SOP (Standard Operating Procedure) for [PROCESS NAME].

- Owner: [ROLE RESPONSIBLE]
- Frequency: [DAILY / WEEKLY / MONTHLY / PER EVENT]
- Trigger: [WHAT STARTS THIS PROCESS]
- Inputs needed before starting: [LIST]

Steps: numbered, specific, no ambiguity. Include decision points (if X, then Y).

Output:
- [DESCRIBE WHAT "DONE" LOOKS LIKE]

Escalation path:
- [WHAT TO DO IF SOMETHING GOES WRONG]

**What makes a good SOP:**
- Write for someone who has never done this before
- Every step has one clear action
- Decision points are explicit, not implied
- The output section defines done — not "complete the task" but "the file is uploaded and the confirmation email is received"
- SOPs are sellable documentation for any business — they demonstrate operational maturity
