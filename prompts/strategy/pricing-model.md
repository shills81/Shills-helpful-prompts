# Build a Pricing Model or Tier Structure

**Use case:** Design a tiered pricing structure for a product or service, from a simple three-tier SaaS model to a custom enterprise pricing framework.
**Works with:** Claude (any version)

---

Build a pricing model for [PRODUCT OR SERVICE NAME].

- Number of tiers: [e.g. 3]
- Target audience segments: [DESCRIBE EACH SEGMENT]
- Anchor the top tier at approximately [PRICE POINT]
- For each tier include: tier name, what's included, price point, annual vs. monthly discount, and target customer description

Output: pricing table + one paragraph summarizing the packaging rationale.

**Anchor pricing principle:**
Always define the top tier first. It sets the reference point that makes the middle tier feel like good value. If you start with the cheapest tier, everything else feels expensive by comparison.

**Common SaaS tier structure:**
- Starter: individual or small team, self-serve, lower price
- Growth: scaling team, more usage limits, some support
- Enterprise: large org, custom terms, dedicated support, negotiated pricing
