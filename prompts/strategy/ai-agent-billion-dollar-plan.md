# Make an AI Chat Agent Worth $1B

**Use case:** Full strategic and technical plan to go from a basic embedded chat agent to a defensible, category-defining product.
**Works with:** Claude (any version)

---

You are a principal product designer, staff engineer, and AI-agent strategist.

The product: [YOUR CHAT AGENT NAME], an AI chat agent inside [YOUR APP NAME].

**Current state:**
- [DESCRIBE CURRENT IMPLEMENTATION — e.g., floating button + panel, direct browser API calls, regex-based action blocks, N-message localStorage history]
- Missing: [LIST WHAT'S NOT BUILT YET — e.g., no streaming, no native tool_use, no attachments, no server proxy, no eval harness, no audit trail]
- Unused assets: [e.g., a prompts module with N prompts never wired to the agent]

**Your job:** Produce the plan to make [YOUR AGENT NAME] worth $1B in five years.

Return:

1. **Category bet** — the one category this agent is creating, in a single sentence. Why is the window open now?

2. **Three never-done-before UX bets** — each a paragraph, each with a testable first experiment that could ship in 2 weeks.

3. **90-day engineering plan** — prioritized list. For each item: why it matters, files or components touched, rough effort (S/M/L), risk, and how you'd measure success.

4. **Moat** — 5 sources of defensibility, each with the mechanism and the first proof point.

5. **Design language** — 10 specific decisions that make this agent visibly different from every other chat product. Not vibes — decisions.

6. **Ops-grade foundation** — minimum viable version of: eval harness, audit trail, permissions, security hardening, cost ceilings.

7. **North-star metric** and 3 supporting metrics, with target numbers at day 90, day 180, and year 1.

8. **Kill criteria** — what would make you shut this down?

Be specific. Cite actual files and components. No generic AI-platform advice.
